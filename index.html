DELETE /nudge/{id}

Delete a nudge (nudgeIdDelete)

Remove a nudge from the system by its ID.

Path Parameters

id (required)
Path Parameter — Unique identifier for the nudge

Responses

204: Nudge deleted successfully

404: Nudge not found

GET /nudge/{id}

Get details of a nudge (nudgeIdGet)

Fetches details of a nudge by its ID.

Path Parameters

id (required)
Path Parameter — Unique identifier for the nudge

Return type

NudgeResponse

Example data

Content-Type: application/json
{
  "image" : "image",
  "scheduledTime" : "scheduledTime",
  "icon" : "icon",
  "scheduledDate" : "2000-01-23",
  "description" : "description",
  "invitationText" : "invitationText",
  "id" : "id",
  "tag" : "tag",
  "title" : "title"
}

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200: Nudge details retrieved successfully

404: Nudge not found

PUT /nudge/{id}

Update a nudge (nudgeIdPut)

Updates an existing nudge's details by its ID.

Path Parameters

id (required)
Path Parameter — Unique identifier for the nudge

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

body Nudge (required)
Body Parameter — Provide updated details of the nudge in JSON format.

Example data

Content-Type: application/json
{
  "image" : "image",
  "scheduledTime" : "scheduledTime",
  "icon" : "icon",
  "scheduledDate" : "2000-01-23",
  "description" : "description",
  "invitationText" : "invitationText",
  "id" : "id",
  "tag" : "tag",
  "title" : "title"
}

Responses

200: Nudge updated successfully

400: Invalid Data

404: Nudge not found

POST /nudge

Create a new nudge (nudgePost)

Creates a new nudge in the system.

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request Body

body Nudge (required)
Body Parameter — Provide details of the nudge to be created in JSON format.

Example data

Content-Type: application/json
{
    "image" : "image",
    "scheduledTime" : "scheduledTime",
    "icon" : "icon",
    "scheduledDate" : "2000-01-23",
    "description" : "description",
    "invitationText" : "invitationText",
    "id" : "id",
    "tag" : "tag",
    "title" : "title"
}

Responses

201: Nudge created successfully

400: Invalid request

Nudge

title
String Title of the nudge
tag
String Tag for the event or article related to the nudge
image (optional)
byte[] Image URL or binary data for the nudge cover format: binary
scheduledDate
date Scheduled date for the nudge (format: YYYY-MM-DD) format: date
scheduledTime
String Scheduled time for the nudge (format: HH:MM) format: time
description (optional)
String Detailed description of the nudge
icon (optional)
String Icon URL or data for the nudge preview
invitationText (optional)
String One-line invitation message for minimized nudge view
